# Git Workflows skill

## What it does
The git-workflows skill handles repository operations including clone, checkout, merge, tagging, cherry-pick, status, diff, blame, discard, commit, and log.

## Tools
Key tools:
- `git_command`
- `git_status`
- `git_diff`
- `git_log`
- `git_blame`
- `git_discard_changes`
- `git_manage_branch`
- `git_commit`
- `git_clone`
- `git_manage_tag`
- `git_merge`
- `git_cherry_pick`

## How to use
Use it when you need Qode to:
- inspect repository state,
- review committed or staged changes,
- switch, create, or merge branches,
- manage tags.

## Example usage
Inspect repository state:
```text
Show git status and the last 10 commits.
```

Review changes:
```text
Show staged changes for src/.
```

Manage branches:
```text
Create a feature branch named feat/docs-update from main.
```
